# remove-unused-css


## Описание
Инструмент для поиска и удаления неиспользуемых CSS-классов на вашем сайте.  
Скрипт анализирует реальные CSS-классы, используемые в DOM, позволяет вручную «пробежать» по всем состояниям интерфейса (всплывающие окна, формы, раскрывающиеся списки и т. д.), собрать список используемых селекторов и сгенерировать оптимизированные CSS-файлы.

> **Внимание:** Проект находится в бета-стадии. Возможны нестабильности и несовместимости с некоторыми фреймворками. По обнаруженным багам или предложениям пишите в [Issues](https://github.com/commeta/remove-unused-css/issues).

## Особенности
- Реальное мониторирование использования CSS в браузере
- Генерация отдельных CSS-файлов с сохранением структуры каталогов
- Автоматическая сборка одного минимизированного файла `remove-unused-css.min.css`
- Лёгкая интеграция на любые сайты без сборщиков

## Установка

1. **Скачайте** или **клонируйте** репозиторий в корень вашего веб-проекта:
   ```bash
   git clone https://github.com/commeta/remove-unused-css.git remove-unused-css
   ```

2. **Скопируйте** в него файлы:

   * `remove-unused-css.js`
   * `remove-unused-css.php`
3. **Создайте** в корне сайта папку `remove-unused-css` (если не клонировали весь репозиторий).

## Подключение

Вставьте перед закрывающим тегом `</body>` на всех страницах вашего сайта:

```html
<script src="/remove-unused-css/remove-unused-css.js"></script>
```

## Использование

1. **Обход всех страниц и состояний**
   Откройте сайт в браузере и вручную «пройдите» по всем страницам, включая скрытые блоки: модальные окна, выпадающие меню, табы и т. д. Это необходимо, чтобы скрипт успел засечь все используемые классы.

2. **Генерация файлов**
   После того как вы убедились, что все нужные классы отловлены, на панели в правом нижнем углу нажмите кнопку **«Сгенерировать файлы»**.

3. **Получение результата**
   В папке `remove-unused-css/css/` появятся:

   * отдельные файлы `.css`, повторяющие структуру исходных стилей;
   * объединённый и минимизированный файл `remove-unused-css.min.css`.

4. **Финальная замена**

   * Уберите из HTML подключение скрипта `remove-unused-css.js`.
   * Замените ваши раскомментированные `<link>` на один `<link>` к `remove-unused-css/css/remove-unused-css.min.css`.

## Структура каталогов после генерации

```
remove-unused-css/
├── css/
│   ├── vendor/
│   │   └── framework.css
│   ├── components/
│   │   └── buttons.css
│   └── remove-unused-css.min.css
├── remove-unused-css.js
└── remove-unused-css.php
```

## Рекомендации

* Тщательно обрабатывайте все состояния интерфейса и разрешения экрана.
* Перед массовым развёртыванием сохраните оригинальные файлы стилей.
* При нестабильном поведении отдельных фреймворков оставьте их оригинальные `.css` в проекте до устранения проблем.
